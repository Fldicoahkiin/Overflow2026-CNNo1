# Simulated Trading Sandbox / 模拟交易沙盒

> 开发者在沙盒中测试策略，不影响真实仓位

## 解决什么问题

开发者在测试新策略时面临两难：测试网数据可能不够真实，主网测试成本太高且风险大。Simulated Trading Sandbox 提供一个模拟交易环境，使用真实的市场数据（OracleSVI）但虚拟的资金，让开发者安全地测试策略逻辑、调试交易流程和验证风控规则，不影响真实仓位和资金。

## 核心功能

- 模拟 Predict 环境：完整模拟 mint/redeem/supply 操作
- 真实市场数据：使用测试网或主网的实时 OracleSVI 数据
- 虚拟资金系统：初始分配虚拟 dUSDC，追踪虚拟盈亏
- 调试工具：逐步执行策略、查看中间状态、设置断点

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| OracleSVI | 使用真实预言机数据驱动模拟 |
| predict-server API | 获取真实市场数据 |
| Predict 合约逻辑 | 模拟合约定价和结算 |
| Sui 事件 | 学习事件结构用于模拟 |

## 实现方案

### SDK 核心
- 沙盒运行时：本地模拟 Predict 合约状态的轻量级引擎
- 数据桥接：从 predict-server 获取实时 OracleSVI 注入沙盒
- 虚拟状态管理器：维护虚拟的仓位、余额和结算记录
- 策略调试器：支持断点、单步执行、状态检查
- 结果比较器：对比沙盒结果与真实合约预期结果

### 前端 / 后端
- 技术栈：React + TypeScript + Node.js
- 沙盒控制台：策略执行状态、虚拟仓位和盈亏
- 调试面板：断点设置、变量检查、执行日志
- 性能监控：策略执行延迟、模拟准确度
- 一键迁移：测试通过后一键切换到真实合约

## 技术架构

开发者编写策略 → 加载到沙盒运行时 → 数据桥接注入实时 OracleSVI → 策略在虚拟环境中执行 → 虚拟状态管理器追踪所有变化 → 调试器支持逐步检查 → 结果比较器验证准确性。
