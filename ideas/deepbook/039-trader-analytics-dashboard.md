# Trader Analytics Dashboard / 交易者分析仪表盘

> 追踪链上交易者地址，统计胜率、盈亏、持仓风格

## 解决什么问题

Predict 市场缺乏透明的交易者分析工具，用户难以评估自身或其他交易者的表现。Trader Analytics Dashboard 追踪链上交易者地址，自动统计每个地址的胜率、盈亏、持仓风格（保守/激进/套利），帮助用户了解自己和市场中其他参与者的交易行为模式，为决策提供数据支持。

## 核心功能

- 地址追踪：输入 Sui 地址即可查看完整交易历史
- 核心指标：胜率、总盈亏、平均收益/亏损、夏普比率
- 持仓风格分析：分类为保守型/激进型/套利型/做市型
- 对比分析：与市场平均水平或指定地址对比

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| 链上 mint/redeem 事件 | 追踪所有交易行为 |
| PredictManager | 关联地址与仓位 |
| OracleSVI | 评估交易时的市场环境 |
| predict-server API | 获取历史交易数据 |

## 实现方案

### Bot / Keeper / 服务端
- 事件索引器：全量索引 Predict 的 mint 和 redeem 事件
- 交易匹配器：将 mint 和 redeem 配对为完整交易记录
- 指标计算器：胜率、PnL、Sharpe、最大连胜等
- 风格分类器：基于持仓时间、仓位大小、行权价选择模式进行分类
- 排名引擎：全局排名和分维度排名

### 前端 / 后端
- 技术栈：React + Recharts + Node.js + PostgreSQL
- 地址搜索和概览页：核心指标和交易摘要
- 盈亏曲线：时间维度的累计 PnL 变化
- 交易热力图：按行权价和到期日的交易分布
- 对比面板：多地址并排对比

## 技术架构

事件索引器全量采集 mint/redeem → 交易匹配器配对完整交易 → 指标计算器统计核心指标 → 风格分类器判断交易风格 → 存入 PostgreSQL → 前端通过 API 查询展示。
