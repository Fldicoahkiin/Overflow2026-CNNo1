# Whale Tracker / 链上大户追踪

> 追踪大额 Predict 交易，实时推送 Whale Alert

## 解决什么问题

大额交易（Whale 交易）往往预示着市场方向变化或知情交易者的行为，但 Predict 市场缺少实时的大户追踪工具。Whale Tracker 监控所有超过阈值的大额 Predict 交易，实时分析交易意图并推送 Whale Alert。帮助交易者及时发现大户动向，作为交易决策的重要参考信号。

## 核心功能

- 实时监控超过阈值的大额 mint/redeem 交易
- 大户画像：追踪大户地址的持仓风格和历史胜率
- 多通道告警：Telegram/Discord/Web 推送 Whale Alert
- 大户排行榜：按持仓规模和活跃度排名

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint 事件 | 检测大额开仓 |
| predict::redeem 事件 | 检测大额赎回 |
| PredictManager | 追踪大户持仓 |
| 链上事件 | 大户地址行为分析 |

## 实现方案

### Bot / Keeper / 服务端
- 事件监听器：实时订阅所有 mint/redeem 事件
- 金额过滤器：筛选超过阈值的大额交易
- 大户分析器：追踪大户地址的历史行为和胜率
- 意图推断器：根据交易参数推断大户意图（方向性/套利/做市）
- 告警分发器：多通道实时推送
- 大户数据库：长期追踪大户地址的所有交易

### 前端 / 后端
- 技术栈：React + WebSocket + Node.js + PostgreSQL
- Whale Alert 面板：实时大额交易流
- 大户详情页：持仓、历史、胜率分析
- 告警配置：阈值设置和通道选择
- 统计图表：大户活跃度趋势和方向偏好

## 技术架构

事件监听器实时捕获 mint/redeem → 金额过滤器筛选大额交易 → 大户分析器查询历史行为 → 意图推断器分析交易目的 → 告警分发器多通道推送 → 前端实时展示 Alert 流和大户画像。
