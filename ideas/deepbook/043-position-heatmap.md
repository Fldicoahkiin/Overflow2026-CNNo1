# Position Heatmap / 仓位热力图

> 可视化所有链上仓位的行权价和到期日分布

## 解决什么问题

Predict 市场的所有仓位都在链上可见，但缺少直观的工具展示整体市场仓位的分布情况。交易者需要了解哪些行权价和到期日最受关注、市场整体偏向看涨还是看跌、资金集中在哪里。Position Heatmap 将所有链上仓位可视化为热力图，按行权价和到期日维度展示市场分布，帮助交易者发现拥挤交易和冷门机会。

## 核心功能

- 全市场仓位热力图：行权价 × 到期日维度的仓位量分布
- 多指标切换：按仓位数量、金额、方向（涨/跌）着色
- 时间演变：热力图随时间的动态变化动画
- 自定义筛选：按地址、金额范围、方向过滤

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::mint 事件 | 采集所有开仓数据 |
| 链上仓位状态 | 获取活跃仓位详情 |
| PredictManager | 关联仓位与行权价/到期日 |
| predict-server API | 获取聚合仓位数据 |

## 实现方案

### Bot / Keeper / 服务端
- 仓位索引器：全量索引所有 mint 事件和活跃仓位
- 数据聚合器：按行权价和到期日聚合仓位数量和金额
- 热力图生成器：将聚合数据转为热力图矩阵
- 时间序列存储：定期快照仓位分布用于动态展示

### 前端 / 后端
- 技术栈：React + D3.js / ECharts + WebGL
- 热力图面板：交互式二维热力图，悬停显示详情
- 时间动画：播放仓位分布随时间的变化
- 筛选控制面板：金额范围、方向、到期日过滤
- 趋势指标：市场整体看涨/看跌比、最大痛点分析

## 技术架构

仓位索引器采集所有 mint 事件 → 数据聚合器按行权价/到期日统计 → 热力图生成器创建矩阵 → 时间序列存储记录变化 → 前端渲染交互式热力图 → 用户可筛选和回放。
