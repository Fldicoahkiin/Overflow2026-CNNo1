# Settlement Leaderboard / 结算排行榜

> 按周期统计最准确的预测者，按盈亏排名

## 解决什么问题

Predict 市场缺少一个公认的、基于链上数据的排行榜来展示最佳预测者。现有排行榜通常是项目方自建的，缺乏透明度和可信度。Settlement Leaderboard 直接从链上结算事件中统计，按不同周期（日/周/月）排名最准确的预测者，按真实盈亏排序，所有数据可验证，成为 Predict 生态的权威排名。

## 核心功能

- 多周期排行榜：日榜/周榜/月榜/全时榜
- 双维度排名：按胜率排名和按盈亏金额排名
- 预测者详情页：历史交易、胜率趋势、持仓风格
- 链上验证：每个排名数据都可追溯到链上交易

## 使用的 DeepBook / Sui 能力

| 能力 | 用途 |
|------|------|
| predict::redeem 事件 | 追踪结算结果和盈亏 |
| predict::mint 事件 | 追踪开仓行为 |
| PredictManager | 关联地址与交易记录 |
| 链上事件 | 全量数据来源保证可信 |

## 实现方案

### Bot / Keeper / 服务端
- 结算事件索引器：全量索引所有 redeem 事件
- 盈亏计算器：对比 mint 成本和 redeem 收益计算每笔盈亏
- 排名引擎：按多维度和周期计算排名
- 趋势分析器：追踪排名变化和胜率趋势
- 数据验证器：提供链上验证接口

### 前端 / 后端
- 技术栈：React + Next.js + PostgreSQL
- 排行榜主页：多周期 Tab 切换、双维度排名
- 预测者详情页：交易历史、盈亏曲线、能力雷达图
- 排名变化动画：名次上升/下降可视化
- 链上验证入口：点击查看原始链上交易

## 技术架构

结算事件索引器持续采集 redeem 事件 → 盈亏计算器匹配 mint 成本 → 排名引擎按多维度计算 → 存入 PostgreSQL → 前端通过 API 获取排行数据 → 每个数据点链接到链上原始交易。
